>> provided screen shoots to fix skipped line in pixel size 32 and 16

Found and fixed the bug. In step_scan_draw's hex branch, 
odd-column rows computed their row index as round((y - rowOffset) / rowHeight), 
subtracting a half-row before rounding. Since y (from the accumulating my.scanY += yStep) 
drifts by tiny float error around each exact row boundary, 
that subtraction put the rounding right on a tie (k - 0.5), 
and any negative drift flipped it down a row — skipping that row entirely for odd columns. 
I verified numerically that this reproduces for pixelSize 32 and 16 
(and others) and always skips 1-3 rows over a typical canvas height.

The fix (step_scan.js:63-72): round the unshifted y / rowHeight 
first (always near a full integer with a safe margin), 
then add the half-row offset afterward instead of before rounding. 
Confirmed via simulation this produces zero missing rows across all tested pixel sizes.
